はじめに

このツールは、石垣などの測量データを解析・可視化するためのブラウザアプリです。 オブジェクトの輪郭データから幅・高さ・面積・傾きなどの属性を自動計算し、 それらを色分けして立面図として可視化、CSV・GLB・SVGとして書き出します。

石垣BIMのワークフローでは、 ①フォトグラメトリで3Dモデル化 → ②クラスタリングで石の輪郭を抽出（DXF）→ ③このアプリで属性計算・色分け → ④報告書用の図やデータを書き出し という流れの、③④を担います。

すべての処理はあなたのブラウザ内で完結します。データがサーバーへ送信されることはありません。 インターネット接続がなくても（一度ページを開けば）動作します。

クイックスタート

まず何ができるか触ってみたい場合の最短手順です。所要2〜3分。

1サンプルを表示する

アプリを開くと、内蔵のサンプル石垣が自動で表示されます（何も読み込まなくてOK）。 マウスでドラッグして視点を回してみてください。

2色分けする

左パネルの「3. 色分けする」で属性（例: Width＝幅）を選ぶと、 オブジェクトが値に応じてグラデーション表示されます。凡例とヒストグラムが下に出ます。

3傾きラインを見る

「4. 傾きラインを描画する」の「傾きラインを表示」にチェックを入れると、 各オブジェクトに傾きを示す線が引かれます。

4書き出す

「7. 書き出す」で「立面図をSVGで保存」を押すと、 いま見えている色分け・凡例つきの立面図がベクター画像として保存されます。

ビュー右上のボタン

  • 正面 — 立面図SVGと同じ向き（壁の正面）から見ます。書き出し前の構図確認に便利です。
  • 全体 — モデル全体が画面に収まるように表示します。
  • ☀ ライト / 🌙 ダーク — 背景色を切り替えます。投影や印刷、スクリーンショット向け。

ステップ別ガイド

左パネルは上から順に番号のついたステップになっています。 各ステップのタイトルをクリックすると開閉できます。 閉じていても、タイトル右側に現在の設定状態（例: Width(幅)）が薄く表示されます。

1. モデルを読み込む

解析したい3Dデータを読み込みます。ボタンから選ぶか、ビューポート（右の3D表示領域）へ ファイルをドラッグ＆ドロップしても読み込めます。

  • GLB / DXF — メインのジオメトリ。DXFは読み込むと同時に属性が自動計算されます。
  • フォトグラメトリモデル（背景） — OBJ+MTL+テクスチャ画像を重ねて表示できます。 位置合わせの参考にする背景用です。ファイルをまとめて選択するか、フォルダごと選択します。
  • データ名 — このデータの名前。ファイル名が自動で入りますが、 変更できます（例: 本丸東面_2026調査）。この名前は書き出すCSVに記録されます （書き出しの使い分けを参照）。
背景モデルも重ねる場合は、DXF/GLBを先に読み込んでから背景OBJを読み込んでください。 先に読み込んだ方の座標を基準に位置合わせが行われます。

2. 属性データ（GLB利用時）

GLBモデルに外部のCSVで属性を紐づける場合に使います。 CSVのID列とモデルのオブジェクト名を対応させて属性を読み込みます。 DXFの場合は属性が自動計算されるため、このステップは基本的に不要です。

3. 色分けする

属性の値に応じてオブジェクトに色をつけます。

  • 数値属性（幅・傾きなど）— グラデーションで表示。 配色プリセット（赤→青など）、色の反転、範囲の手動指定ができます。
  • テキスト属性（石材種別など）— カテゴリごとに色分け。 種類が多い場合は「上位7カテゴリのみ色分け」で主要なものだけを目立たせ、 残りをグレーの「その他」にまとめられます。

属性を選ぶと自動で反映されます。凡例とヒストグラム（値の分布）が画面下に表示されます。

4. 傾きラインを描画する 石垣分析

各オブジェクトの外接矩形の水平軸に沿った中心線を描き、その傾きを線の色で表します。 石の水平方向の連続性が一目で分かります。

  • 色 — 傾きに応じたグラデーション（初期設定は 0°＝青、±45°＝赤）。 中央色・両端色は自由に変更できます。
  • 太さ — 20〜100mm（実寸）で調整。ズームしても実際の寸法として表示されます。

5. フィルタで絞り込む

条件に合うオブジェクトだけを表示（または半透明・非表示）にします。 「幅が0.5m以上」「自己交差している輪郭を除外」など、数値属性に対する範囲条件を 複数組み合わせられます（すべての条件を満たすものが対象）。

6. ラベルを表示する

各オブジェクトにID・属性値などを文字で表示します。表示する内容と文字サイズを選べます。 オブジェクトが多い場合、遠くのラベルは自動的に間引かれます（動作を軽く保つため）。

7. 書き出す

結果を3つの形式で書き出せます。用途別の詳しい説明は 書き出しの使い分けを参照してください。

  • 属性をCSVで保存 — 全属性を表形式で。BIツールやPythonでの分析用。
  • ID付きGLBで保存 — 3Dモデルとして。他のツールでの再利用用。
  • 立面図をSVGで保存 — 報告書用のベクター図。含める要素（凡例・スケールバー・ ラベル・傾きライン）や輪郭線の太さを選べ、塗りなしの線画モードもあります。

データの準備

DXF（推奨）

オブジェクトの輪郭を表す閉じたポリラインで構成されたDXFファイル。読み込むと属性が自動計算されます。 以下の2系統に対応しています。

  • 閉じた3Dポリライン（クラスタリングソフトの出力形式）
  • 2Dポリライン（Rhinoなどの出力形式）— 傾いた座標系（OCS）は自動でワールド座標に変換されます

レイヤー名の末尾の数字がIDとして使われます（例: Stone_123 → 123）。

GLB + CSV

Rhino/Grasshopperなどから書き出した3Dモデル（GLB）に、 別途用意したCSVの属性を紐づける方式。CSVにはID列と属性列が必要です。

OBJ（背景モデル）

フォトグラメトリで作成したテクスチャ付きモデルを背景として重ねられます。 OBJ本体・MTL・テクスチャ画像をまとめて選択するか、 フォルダごと選択してください。テクスチャは自動で適切なサイズに縮小されます。

DXFやGLBが実際の測量座標（平面直角座標など、値がとても大きい座標）を持っていても、 アプリが自動的に原点を寄せて正しく表示します。座標を事前に加工する必要はありません。

書き出しの使い分け

CSV — データ分析用

計算した全属性を表形式で書き出します（Excel・BIツール・Pythonで開けます）。 文字コードはExcelでも文字化けしないUTF-8（BOM付き）です。

ステップ1でデータ名を設定しておくと、CSVの全行に Dataset 列として その名前が入ります。複数の調査データを横断的に分析するときの「どのデータ由来か」の 識別に使えます。たとえばPythonの pandas では次のように結合できます。 

import pandas as pd, glob
df = pd.concat([pd.read_csv(f) for f in glob.glob("*.csv")])
df.groupby("Dataset")["Width"].describe()   # データ別に幅を集計

GLB — 3Dモデルとして再利用

オブジェクト名（ID）を保持したGLBを書き出します。Blender・Rhinoなど他のツールで 開いて再利用できます（DXF→GLBの変換としても使えます）。法線の向きは自動的に揃えられます。

SVG — 報告書用の立面図

モデルを正面から見た立面図を、拡大しても劣化しないベクター画像（SVG）で書き出します。 Illustratorなどで後から編集できます。書き出し時に以下を選べます。

  • 含める要素 — 凡例・スケールバー・ラベル・傾きラインを個別にオン/オフ。 3D表示のオン/オフとは独立しているので、「画面には出さないがSVGには入れる」もできます。
  • 輪郭線の太さ — mm実寸で指定（0にすると輪郭線なし）。
  • 塗りつぶしなし（線画）モード — オブジェクトを塗らず輪郭線だけの図に。実測図・トレース図風の出力に。

属性を追加して読み込み直す（GLB + CSV の往復）

調査で得た独自の情報（石材種別、加工痕の有無、劣化度など）をオブジェクトごとに付けて 色分けしたい場合は、次の往復ワークフローが使えます。

  1. DXFを読み込み、ID付きGLBとCSVを書き出します。 この時点でIDが自動採番され、GLBとCSVは同じIDで対応が取れています。
  2. 書き出したCSVをExcelなどで開き、任意の属性列を追加します （例: StoneType 列に「切込接」「打込接」などを記入）。 ID列と行の対応は崩さないようにしてください。
  3. アプリで手順1のGLBを読み込み、続いて編集したCSVを読み込みます。 GLBのオブジェクト名（ID）とCSVのID列が突き合わされ、追加した属性が反映されます。
  4. 「3. 色分けする」で追加した属性を選べば、独自情報で色分けできます。 テキスト属性ならカテゴリ色、数値属性ならグラデーションになります。
CSVを編集する際はID列の値を変更しないでください。 IDがGLBのオブジェクト名と一致しなくなると、その行の属性は紐づけられません。 追加するのは新しい列だけにするのが安全です。

困ったときは

ファイルを読み込めない・表示されない

  • DXFは閉じたポリラインである必要があります。線分（LINE）や開いた線では読み込めません。
  • GLBは .glb 形式（バイナリ)にしてください。外部バッファを参照する .gltf は未対応です。
  • 読み込み中に別のファイルを読もうとすると「完了までお待ちください」と出ます。少し待ってから操作してください。

傾きラインや色分けが反映されない

  • 傾きラインはDXFやサンプルなど、輪郭データを持つモデルで利用できます。
  • フィルタで除外したオブジェクトには傾きラインも引かれません（意図した連動です）。

全体表示で余白が大きい／モデルが小さい

ビュー右上の「全体」ボタンを押すと、余白5%でちょうど収まる大きさに調整されます。 斜めから見ている場合も、モデルが画面の中央に来るように配置されます。

操作しても反応がないとき（上級者向け）

ブラウザの開発者ツール（F12）を開き、Console（コンソール）タブに 赤いエラーが出ていないか確認すると原因の手がかりになります。 不具合を報告する際は、そのエラー文と、使っている版（画面のバージョン表記）を添えると解決が早まります。

データの安全性について

読み込んだデータはブラウザの中だけで処理され、どこにも送信されません。 ページを閉じるとデータは消えます（自動保存はされません）。必要な結果は書き出して保存してください。
