# Geometry Attribute Colorizer — 開発ガイド

石垣調査（石垣BIM）ワークフローのための、ジオメトリ属性計算・色分け可視化ツール。
このファイルはClaude Codeがこのリポジトリで作業する際の前提とルールを記述する。

> 注: `CLAUDE_blender_addon_OLD.md` は、このツールの前身である
> Blenderアドオン版の記録。**現在は凍結されており、参照する必要はない。**

## プロジェクトの性質

- **単一HTMLファイルで完結する構成**。`index.html` にJS/CSSがすべて含まれる
- 外部依存はCDNの Three.js (r128) と PapaParse (5.4.1) のみ。ビルド工程は無い
- すべての処理がブラウザ内で完結し、データを外部送信しない（設計上の重要な約束）
- GitHub Pagesで公開。`index.html` を差し替えれば即公開される
- 想定規模は最大1万オブジェクト

## 開発時の必須ルール

### 1. 計算ロジックを変更したらテストを再抽出して実行する

テストは `index.html` から実関数を抽出した `tests/extracted.js` に対して実行する。
そのため**コードを変更したら再抽出が必要**。

```bash
node extract_for_tests.js   # index.html から tests/extracted.js を再生成
node tests/run_all.js       # 全テストを一括実行（95件）
```

個別に実行する場合:

```bash
cd tests
node test_suite.js        # 幾何計算・色計算の単体テスト（40件）
node test_rectline.js     # 傾きラインの計算（11件）
node test_palette.js      # カテゴリ色パレット（3件）
node test_fitview.js      # 全体表示のフィット計算（12件）
node test_winding.js      # GLB法線の巻き順（3件）
node test_adversarial.js  # 異常入力への耐性（12件）
node test_rhino_dxf.js    # Rhino形式DXFの読み込み（9件）
node integration_test.js  # 実データでの統合テスト（5件）
```

**全テストが通ることを確認してからコミットする。**

### 2. UI・描画の変更はブラウザでの手動確認が必要

テストは計算ロジックしかカバーしていない。過去に「計算は正しいが、
メッシュへの保存時にキーを1つ入れ忘れて3D表示だけ動かない」バグが発生している
（傾きラインの `rectLine.n` の入れ忘れ）。
UI・3D描画・SVG出力を変更したら、必ずブラウザで実際に動かして確認すること。

### 3. バージョン管理

- セマンティックバージョニング（メジャー.マイナー.パッチ）
- バージョンは `index.html` の `APP_VERSION` 定数と `help.html` の
  `HELP_VERSION` 定数の**両方**を更新する
- リリース時は `CHANGELOG.md` に追記し、`git tag vX.Y.Z` を打つ

## 設計上の重要な決定（変更しないこと）

- **RectAngleの正方形問題には対応しない**。Width≒Heightのオブジェクトで角度が
  ±45°付近で不安定になる件。実データで実害がないため対応不要と判断済み
- **座標の原点シフト（worldOffset）は必須**。実測座標（平面直角座標など）を
  そのまま扱うとfloat32精度が破綻するため、読み込み時に原点付近へ寄せている
- **傾きラインは面から Flatness 分だけ浮かせる**。オブジェクトは完全な平面ではないため、
  面上に置くと凹凸に埋まって線が途切れる
- **法線の向きは「壁全体の平均法線」で統一する**。個々の輪郭の巻き方向はバラバラなので、
  平均法線を基準に揃える（傾きライン・SVG立面図・GLB書き出しで共通の考え方）
- **全体フィットはリセンタリング方式**。距離の二分探索＋画面内平行移動で余白を
  均等配分する。単純な対称フィットでは斜め視点で奥側に余白が残るため

詳しい設計意図は `SPECIFICATION.md` を参照。**設計判断を変更する際は必ずそこも更新する。**

## 用語の統一

- UI表示・ヘルプ本文の総称は「**オブジェクト**」を使う（汎用ツールとしての立ち位置）
- 「石」「石垣」は、石垣調査という具体的文脈を説明する箇所でのみ使う
- 傾きライン（旧称: 通り線 / 回転角ライン）— 現在の正式名称は「傾きライン」

## コーディングスタイル

- コメント・UI文言はすべて日本語
- コメントは**「何をしているか」ではなく「なぜそうするか」を書く**
- 既存のスタイル（インデント2、セミコロンあり）に合わせる
- CSS変数（`--accent` など）を使い、ダーク/ライト両テーマで破綻しないようにする
- 新しい外部ライブラリは原則追加しない（単一ファイル・低依存の方針）
- ステータス表示は左アクセントバーの「通知」、ボタンは面色ありの「操作」と
  視覚的な役割を分けている。この区別を崩さない
